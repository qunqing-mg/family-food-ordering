---

## 一、项目概述

面向家庭成员的点餐小程序。任何成员可发布菜品、创建分类、编辑菜单；家人浏览分类和菜品、加入购物车、提交订单。类似"家庭版小美团"。

### 核心场景

```
妈妈打开小程序 → 创建家庭"幸福一家" → 邀请爸爸和儿子加入
妈妈添加分类：主食、炒菜、汤、饮品
妈妈发布菜品：红烧肉¥25、番茄蛋汤¥8 ...
爸爸打开小程序 → 看到今日菜单 → 点红烧肉+米饭 → 加入购物车 → 下单
妈妈看到订单通知 → 知道要做什么菜 → 采购食材 → 上菜 → 订单完成
```

## 三、功能清单

### 3.1 用户系统（微信登录）

| 序号 | 功能 | 说明 |
|------|------|------|
| U1 | 微信一键登录 | `wx.login()` 获取 code → 后端换取 openid → 自动注册/登录 |
| U2 | 用户信息 | 微信头像、昵称自动同步 |
| U3 | 个人中心 | 我的订单列表、所在家庭信息 |

### 3.2 家庭管理

| 序号 | 功能 | 说明 |
|------|------|------|
| F1 | 创建家庭 | 输入家庭名称，生成6位邀请码 |
| F2 | 加入家庭 | 输入邀请码加入，创建者自动通过 |
| F3 | 成员列表 | 查看家庭所有成员 |
| F4 | 退出/解散 | 成员可退出；创建者可解散（删除全部数据） |

### 3.3 菜品分类

| 序号 | 功能 | 说明 |
|------|------|------|
| C1 | 新增分类 | 名称 + 排序值（主食/炒菜/汤/饮品...） |
| C2 | 编辑/删除分类 | 删除分类时其下菜品一并下架 |

### 3.4 菜品管理

| 序号 | 功能 | 说明 |
|------|------|------|
| D1 | 发布菜品 | 名称、图片（拍照/相册）、价格、分类、描述 |
| D2 | 编辑/下架 | 可修改信息或下架（售罄） |
| D3 | 菜品列表 | 左侧分类栏 + 右侧菜品网格，支持按分类筛选 |

### 3.5 购物车与下单

| 序号 | 功能 | 说明 |
|------|------|------|
| S1 | 加入购物车 | 选择菜品 + 数量，加减操作 |
| S2 | 购物车浮层 | 底部栏显示总价和件数，点击弹出详情 |
| S3 | 提交订单 | 确认菜品列表+总价 → 生成订单 |
| S4 | 订单状态 | 待接单 → 已接单 → 已完成 / 已取消 |
| S5 | 订单通知 | 下单后发布者收到通知（小程序订阅消息） |

### 3.6 订单管理

| 序号 | 功能 | 说明 |
|------|------|------|
| O1 | 订单列表 | 分"我下的单"和"我收到的单"两个 Tab |
| O2 | 接单/完成/取消 | 发布者可操作订单状态变更 |
| O3 | 订单详情 | 菜品清单 + 数量 + 单价 + 总价 |

## 四、数据库设计（7张表）

```
user
│ id, openid, nickname, avatar_url, create_time

family
│ id, name, invite_code, creator_id, create_time

family_member
│ id, family_id, user_id, join_time

category
│ id, family_id, name, sort_order, create_time

dish
│ id, family_id, category_id, name, image_url, price, description, status(1上架/0下架), create_time

order
│ id, family_id, user_id, total_amount, status(0待接单/1已接单/2已完成/3已取消), create_time, update_time

order_item
│ id, order_id, dish_id, dish_name, dish_image, quantity, price
```

> `order_item` 冗余存 dish_name / dish_image 防止菜品被删除后订单信息丢失。

## 六、接口设计（核心接口）

### 用户
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/wx-login` | 微信登录（code → token） |

### 家庭
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/family` | 创建家庭 |
| POST | `/api/family/join` | 加入家庭（邀请码） |
| GET | `/api/family/members` | 成员列表 |
| DELETE | `/api/family/member/{id}` | 踢出成员（创建者） |

### 分类
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/category` | 新增分类 |
| GET | `/api/category/list` | 分类列表 |
| PUT | `/api/category/{id}` | 编辑分类 |
| DELETE | `/api/category/{id}` | 删除分类 |

### 菜品
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/dish` | 新增菜品 |
| GET | `/api/dish/list` | 菜品列表（可按分类筛选） |
| PUT | `/api/dish/{id}` | 编辑菜品 |
| PUT | `/api/dish/{id}/status` | 上架/下架 |

### 订单
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/order` | 提交订单 |
| GET | `/api/order/list` | 订单列表（我的/收到的） |
| PUT | `/api/order/{id}/status` | 接单/完成/取消 |

## 七、技术要点

| 维度 | 方案 |
|------|------|
| 微信登录 | `wx.login()` → 后端 `code2Session` → 生成 JWT Token |
| 图片上传 | uni-app `chooseImage` → 后端接收 → 存本地/OSS → 返 URL |
| 购物车 | 前端 Pinia 管理，刷新保留（`uni.setStorageSync`） |
| 多家庭隔离 | 所有查询接口带 `family_id`，中间件校验当前用户是否在该家庭 |
| 数据库 | MySQL 8，每家庭数据逻辑隔离 |
